Git Reference

Информация

git config --global user.name "Your Name"
git config --global user.email "your_email@gmail.com" - установка имени и электронной почты.

git config --system - настройки на уровне системы.
git config --global - настройки на уровне пользователя.
git config [--local] - настройки на уровне проекта.
git config --unset - удаление поля в файле настройках.
git config --list - cписок установленных настроек git.
git help - справка по командам git.
git help command - информация по указанной команде.

.git - каталог git-репозитория.
repo.git - чистый репозиторий (не содержащий рабочего каталога), представляющий каталог .git
.gitignore - файл указывающий исключения для git (node_mudules/ - для папок).
.gitconfig - пользовательский файл конфигурации git (находится в домашней директории пользователя /home/username/).
.git/config - локальный файл конфигурации git.
HEAD - указатель на текущию ветку, текущего коммита.
origin - имя удалённого репозитория по умолчанию.
master - имя ветки по умолчанию.

Основное

git init - создание репозитория.
git add file - добавление файла в Staging Area.
git add . - добавление всех файлов в Staging Area.
git commit -m "Some comment" - коммит изменений.
git commit --amend -m "Some comment" - добавление изменений в последний коммит.
git status - состояние репозитория.
git log - история коммитов.
git log branchName - история коммитов в указанной ветке.
git log file - история коммитов для указанного файла.
git log --oneline - вывод истории коммитов в сокращённом(компактном) виде.
git log --graph --all - история коммитов с графическим отображением веток.
git show - информация о текущем коммите.
git show hash-коммита - инормация по указанному коммиту.
git reflog - история действий в git.
git reflog branchName - история действий в указанной ветке.
git checkout hash-коммита - откат изменений к указанному коммиту.
git checkout branchName - перейти к последнему коммиту, указанной ветки.
git checkout HEAD file - переключиться на последний коммит указанного файла в репозитории (отмена непроиндексированных изменений в рабочем каталоге).
git reset file - отменяет проиндексированные изменения для указанного файла в Staging Area.
git reset - Сброс Staging Area. Отмена всех проиндексированных изменений в Staging Area.
git diff - разница между изменениями в рабочем каталоге и Staging Area.
git diff HEAD - разница между изменениями в рабочем каталоге и репозитории.
git diff file - разница между изменениями для указанного файла в рабочем каталоге и репозитории.
git diff --staged - разница между изменениями в Staging Area и репозитории.
git diff firstBranch secondBranch - различие между ветками.
git diff hash-коммита-1 hash-коммита-2 - различие между коммитами.
git tag tagName hash-коммита - маркировка коммита именованным тегом.
git tag -d tagMame - удаление тега.
git tag - список назначенных(установленных) тегов.
git rm - удаление файла и помещение изменения об удалении в Staging Area.
git rm -r - удаление каталога и помещение изменения об удалении в Staging Area.
git mv - переименование/перемещение файла и помещение изменения в Staging Area.

Ветки

git branch newBranch - создание новой ветки.
git checkout branchName - переключение на указзанную ветку.
git checkout -b newBranch - создание ветки и переключение на неё.
git branch - список локальных веток.
git branch -v - список локальных веток и текущий коммит, на который указывает текущая ветка.
git branch -a - список всех веток (локальных и удалённых).
git branch -d branchName - удаление ветки.
git branch -D branchName - удаление не смёрженной(слитой) ветки.
git merge branchName - слить указанную ветку в текущую ветку.
git rebase branchName - перебазирование указанной ветки на текущую ветку.

Удалённые репозитории

git clone URL-репозитория [cloned-repo] - клонирование удалённого репозитория и сохранение под указанным именем cloned-repo.
git remote add origin URL-репозитория - добавление удалённого репозитория под именем origin.
git remote - список удалённых репозиториев.
git remote show origin - информация об удалённом репозитории.
git push origin branchName - отправка изменений на удалённый репозиторий.
git pull origin branchName - получение изменений с удалённой ветки и слияние изменений в локальную ветку.
git fetch origin branchName - получение изменений с удалённой ветки (без слияния в локальную ветку).
